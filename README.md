# Security SOAR Automation Lab

## Project Overview

This project demonstrates a Security Orchestration, Automation and Response (SOAR) workflow designed to automate the analysis of security alerts generated by firewall logs.

The system integrates a SIEM platform, a threat intelligence platform, and an automation engine to process security events, enrich them with threat intelligence, and generate automated alerts for security analysts.

The goal of this project is to reduce manual analysis in a Security Operations Center (SOC) environment and improve incident response efficiency.

---

## Workflow Description

### 1. Security Event Generation

Security events are generated by the firewall when authentication failures or suspicious access attempts occur.

These events are forwarded to the SIEM platform for analysis.

---

### 2. SIEM Alert Processing

The SIEM analyzes firewall logs and generates security alerts when suspicious behavior is detected.

The alert data contains important information such as:

- Source IP address
- Alert severity
- Event description
- Timestamp

---

### 3. Automation Trigger

The automation workflow begins when the SIEM sends the alert data to the automation engine.

The workflow extracts relevant indicators of compromise from the alert.

---

### 4. Threat Intelligence Enrichment

The extracted IP address is queried against a threat intelligence database to determine whether the IP is associated with known malicious activity.

Two outcomes are possible:

IP Found in Threat Intelligence Database  
→ The system classifies the event as a malicious alert.

IP Not Found in Threat Intelligence Database  
→ The system classifies the event as suspicious activity.

---

### 5. Decision Engine

A decision engine evaluates the threat intelligence results and determines the appropriate response.

If the IP address exists in the threat intelligence database, the event is flagged as a malicious indicator.

If the IP address is not found, the event is categorized as suspicious but not confirmed malicious.

---

### 6. Automated Alerting

The automation system generates an email notification containing the relevant event information.

The notification includes:

Source IP address  
Event description  
Alert severity  
Threat intelligence result  
Recommended action

These alerts help SOC analysts respond to potential threats quickly.

## Limitations

The current implementation relies on a single threat intelligence source.

If the IP address does not exist in the threat intelligence database, the system classifies the event as suspicious rather than malicious.
